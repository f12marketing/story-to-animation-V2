# Composite Image + Video Generation -> Merge -> Motion Graphics

Five phases: composite scene images -> video clips -> merge -> (optional voiceover) -> motion graphics.

All shots are processed concurrently — for 20 shots the total runtime is roughly
the time for one composite + one video, not 20× that.

## Setup: .env

Same `.env` as Step 3 — if it already exists from running `generate_images.py`, no extra setup needed.

```
FAL_API_KEY=your_fal_api_key_here
R2_ACCOUNT_ID=your_cloudflare_account_id
R2_ACCESS_KEY_ID=your_r2_access_key_id
R2_SECRET_ACCESS_KEY=your_r2_secret_access_key
R2_BUCKET_NAME=your_bucket_name
R2_PUBLIC_URL=https://pub-xxxx.r2.dev
```

If no `.env` is present, the script auto-creates a template and exits.

## Prerequisites

- `shots.json` must exist (from Skill 4)
- `characters.json` and `backgrounds.json` must have `image_url` fields
  (permanent R2 URLs set by `generate_images.py` in Step 3)
- `.env` in project directory with all keys filled in
- `pip install requests boto3`
- FFmpeg installed and in PATH (for merge step)
  - Windows: `winget install ffmpeg`
  - Mac: `brew install ffmpeg`
  - Linux: `sudo apt install ffmpeg`
- Node.js 18+ (for motion graphics step)
  - Windows: `winget install OpenJS.NodeJS`
  - Mac: `brew install node`

## Running the Script

### Per-shot mode (recommended — approve each clip before the next)

```bash
python ~/.claude/skills/generating-composite-and-video/scripts/generate_videos.py --shot shot_001
python ~/.claude/skills/generating-composite-and-video/scripts/generate_videos.py --shot shot_002
# ... etc.
```

### Bulk mode (generate all shots at once, approve at the end)

```bash
cd /path/to/your/project
python ~/.claude/skills/generating-composite-and-video/scripts/generate_videos.py
```

### Regenerate a specific shot (force re-run, ignores existing files)

```bash
python ~/.claude/skills/generating-composite-and-video/scripts/generate_videos.py --regenerate shot_003
```

This deletes `composites/shot_003.png` and `clips/shot_003.mp4` then re-runs that shot.
Useful after editing `veo_prompt` in `shots.json` or when a shot needs to be redone.

## What the Script Does

### Phase 1 — Composite images (parallel, fal-ai/flux-2-pro/edit)

All shots submitted simultaneously via fal.ai queue API.

- `image_urls` = [background `image_url`, character `image_url`(s)] from JSON
- `prompt` = shot `action` + "Pixar-style 3D animation, cinematic 16:9 composition"
- `image_size` = `landscape_16_9`
- Polls `status_url` from submit response until `status: COMPLETED`
- Downloads to `./composites/{shot_id}.png`
- Uploads composite to Cloudflare R2 (retries 3×) for a stable permanent URL to pass to Phase 2

### Phase 2 — Video clips (parallel, fal-ai/vidu/q3/image-to-video/turbo)

All shots submitted simultaneously via fal.ai queue API.

- `image_url` = R2 URL of composite from Phase 1
- `prompt` = `veo_prompt` field from `shots.json`
- `duration` = `3` seconds
- `resolution` = `720p` (default — skips Topaz upscale step)
- `audio` = `true` (ambient audio generated by vidu q3)
- Polls `status_url` from submit response until `status: COMPLETED`
- Downloads to `./clips/{shot_id}.mp4`

The script skips shots where `./clips/{shot_id}.mp4` already exists — safe to re-run.

### Phase 3 — Merge clips (FFmpeg)

After all clips are approved, merge using `merge_clips.py`:

```bash
cd /path/to/your/project
python ~/.claude/skills/generating-composite-and-video/scripts/merge_clips.py
```

- Reads shot order from `shots.json`
- Concatenates `./clips/{shot_id}.mp4` in narrative order using `ffmpeg -f concat`
- Stream copy only (no re-encoding) — completes in seconds
- Output: `./final_animation.mp4`

### Phase 4 — Voiceover mixing (optional, auto-detected)

`merge_clips.py` automatically detects `audio_path` fields in `shots.json`.
If present (populated by `generate_voiceover.py`), it mixes them in and also
produces `./final_animation_voiced.mp4` using FFmpeg adelay + amix.

To add voiceover:
```bash
python ~/.claude/skills/generating-voiceover/scripts/generate_voiceover.py
python ~/.claude/skills/generating-composite-and-video/scripts/merge_clips.py
```

### Phase 5 — Motion graphics (MANDATORY — runs automatically)

After merge is approved, ALWAYS run `apply_motion_graphics.py`:

```bash
cd /path/to/your/project
python ~/.claude/skills/applying-motion-graphics/scripts/apply_motion_graphics.py
```

This adds a branded title card, subtitles, lower thirds, and end screen via Remotion.
Output: `./final_animation_branded.mp4`

On first run, `brand.json` is auto-created. Edit it and re-run.
If the user has provided a logo file, pass it via `logo_path` in `brand.json`.

### Phase 6 — Upscale to 720p (optional — only if video was generated at 360p)

If `VIDEO_RESOLUTION` was `360p`, upscale with Topaz:

```bash
python ~/.claude/skills/generating-composite-and-video/scripts/upscale_video.py
```

Default resolution is now `720p` — this step is skipped unless the user
explicitly asks for upscaling.

## Configurable Settings (top of script)

| Setting | Default | Description |
|---------|---------|-------------|
| `COMPOSITE_MAX_WORKERS` | `5` | Max parallel composite jobs |
| `VIDEO_MAX_WORKERS` | `5` | Max parallel video jobs |
| `VIDEO_DURATION` | `3` | Clip length in seconds (1–16) |
| `VIDEO_RESOLUTION` | `720p` | `360p`, `540p`, `720p`, or `1080p` |
| `VIDEO_AUDIO` | `True` | Generate ambient audio with each clip |
| `FAL_POLL_INTERVAL` | `10` | Seconds between status polls |
| `FAL_MAX_POLLS` | `120` | Max polls per task (120 × 10s = 20 min timeout) |

## API Reference

| Phase | Model | Submit URL |
|-------|-------|-----------|
| Composite | `fal-ai/flux-2-pro/edit` | `https://queue.fal.run/fal-ai/flux-2-pro/edit` |
| Video | `fal-ai/vidu/q3/image-to-video/turbo` | `https://queue.fal.run/fal-ai/vidu/q3/image-to-video/turbo` |

Both use fal.ai queue API: submit returns `status_url` and `response_url` — use
these directly (do not construct from model name, fal.ai strips variant suffixes).

## Regenerating Failed or Rejected Shots

### Option 1: --regenerate flag (recommended)
```bash
python generate_videos.py --regenerate shot_003
```

### Option 2: manual
1. Delete `./composites/{shot_id}.png` and `./clips/{shot_id}.mp4`
2. Optionally update `veo_prompt` in `shots.json`
3. Re-run the script

## Review Gate (MANDATORY)

### Per-shot mode — after EACH shot

After each `--shot` run completes, present this EXACTLY:

```
🎬 Shot [shot_XXX] ready!

- Composite : ./composites/shot_XXX.png
- Video clip: ./clips/shot_XXX.mp4
- Run time  : [X min]

👉 Please review this clip. You can:
  - Approve  -> say "ok" or "next" to generate the next shot
  - Redo     -> say "redo" (re-runs with --regenerate)
  - Adjust   -> edit veo_prompt in shots.json, then say "redo"

⏸️ Waiting for your approval before generating shot_[next].
```

Only run the next `--shot` after explicit user approval.

### Bulk mode — after ALL shots

```
✅ Composite + Video Generation complete!

📋 Summary:
- Composite images: [X] -> ./composites/
- Video clips: [X] -> ./clips/
- Failed: [X] (list any failures)
- Total run time: [X min]

👉 Please review the clips. You can:
  - Approve all -> say "approved" or "merge" to proceed to final merge
  - Regenerate specific shots -> e.g., "redo shot_003"
    (runs --regenerate shot_003 automatically)

⏸️ Waiting for your approval before merging.
```

### Phase 3 (merge) — after clips are approved

Run `merge_clips.py`, then present:

```
🎬 Merge complete!

- Output : ./final_animation.mp4
- Size   : [X] MB
- Clips  : [X] merged in shot order
- Voiced : [./final_animation_voiced.mp4 if voiceover was detected, else "no voiceover"]

👉 Please review final_animation.mp4.
   Say "add voiceover" to generate and mix narration, or proceed to motion graphics.

⏸️ Proceeding to motion graphics (Phase 5)...
```

**IMMEDIATELY after merge approval, run Phase 5 (motion graphics) automatically.**

### Phase 5 (motion graphics) — runs automatically after merge approval

Run `apply_motion_graphics.py` and present the output per the
`applying-motion-graphics` SKILL.md review gate.

**NEVER** mark the pipeline as complete without explicit user approval of
`final_animation_branded.mp4`.
