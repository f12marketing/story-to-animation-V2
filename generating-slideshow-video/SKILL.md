# Slideshow Video — Ken Burns + Transitions (Remotion)

Renders a video from composite images using Remotion. Each image gets a Ken Burns
effect (pan/zoom), with 0.5-second cross-fade transitions between shots.

**Cost: $0** for video generation — only composite images (flux-2-pro) are needed.

## Pipeline Position

```
Step 1: generating-story-from-logline       → story.md
Step 2: extracting-characters-and-backgrounds → characters.json, backgrounds.json
Step 3: generating-character-and-background-images → images in characters/, backgrounds/
Step 4: creating-shot-list                  → shots.json
Step 5a (this skill): generating-slideshow-video → final_animation.mp4  ← FREE video gen
Step 5b (alternative): generating-composite-and-video → final_animation.mp4  ← AI video (~$50+)
Step 6: applying-motion-graphics            → final_animation_branded.mp4
```

This skill covers both Phase 1 (composites) AND Phase 2 (video render) of Step 5,
skipping vidu entirely.

## Prerequisites

- `shots.json` must exist (from Step 4)
- `composites/` folder with `{shot_id}.png` images
  - If composites don't exist yet, run `generate_composites_only.py` first (Phase 1 only)
- Node.js 18+ and npm installed
  - Windows: `winget install OpenJS.NodeJS`
  - Mac: `brew install node`
  - Linux: `sudo apt install nodejs npm`
- `pip install requests boto3` (for composites phase)
- `.env` with `FAL_API_KEY`, `R2_*` keys (for composites phase only)

## Running

### Phase 1 — Generate composite images only (if not already done)

```bash
cd /path/to/your/project
python ~/.claude/skills/generating-slideshow-video/scripts/generate_composites_only.py
```

This generates `composites/{shot_id}.png` for all shots using fal-ai/flux-2-pro/edit.
Skip this if composites already exist.

### Phase 2 — Render slideshow video from composites

```bash
cd /path/to/your/project
python ~/.claude/skills/generating-slideshow-video/scripts/generate_slideshow.py
```

Optional flags:
```bash
--output  final_animation.mp4   # default: final_animation.mp4
--fps     30                    # default: 30
--transition 15                 # cross-fade frames, default: 15 (0.5s at 30fps)
```

## What the Script Does

1. Reads `shots.json` — uses `shot_id` and `duration_seconds` per shot
2. Copies `composites/{shot_id}.png` into `slideshow_project/public/composites/`
3. Generates `composition-data.json` with shot durations and image paths
4. Sets up Remotion project from templates
5. Runs `npm install` (first run only, ~1-2 min)
6. Runs `npx remotion render` → `final_animation.mp4`

## Ken Burns Variants (cycles through shots)

| Variant | Effect |
|---------|--------|
| `zoom-in` | Scale 1.0 → 1.15 |
| `zoom-out` | Scale 1.15 → 1.0 |
| `pan-right` | Pan left→right at 1.1× scale |
| `pan-left` | Pan right→left at 1.1× scale |
| `pan-up` | Pan bottom→top at 1.1× scale |

## Cross-fade Transitions

Default: 15 frames (0.5s at 30fps). Each shot fades in over the transition duration
and fades out at its end, overlapping the next shot's fade-in for a smooth dissolve.

## Output Files

| File | Description |
|------|-------------|
| `final_animation.mp4` | Rendered slideshow video |
| `slideshow_project/` | Remotion project (can be deleted after render) |
